Graphite

1. Basics

In graphite variables are declared via the var keyword. Ex: var x = 3; if you want to make the variables type non changeable. Use the strict keyword. Ex: strict var y = "hello"; y = 3 will give an error because 3 and "hello" are not of the same type. if you want to make a variable global. use the global keyword. if(1==1){global strict var x = 0;} Terminal.IO.println(x); will work because the global keyword was used. There are 4 main data types: Int,String,Double,Bool. If you want to free up variables for some reason, use the free(varname). You can use the const keyword to create a constant. The global and strict keyword can also be used before const.

2. Input/Output

To output text into the terminal, use the command Terminal.IO.print() or Terminal.IO.println() to make a newline after the text. You can put as much arguments as you want in Terminal.IO.print() or Terminal.IO.println(), it will print it out. for taking user input, use Terminal.IO.input(varname) where varname is the name of the variable you want the input to be assigned to. To clear the terminal use the Terminal.clear(), clear doesnt accept any arguments.

3. Functions

To create a function use the fn keyword. Ex: fn test(){Terminal.IO.println("test");} Graphite supports recursion. So you can call functions from inside. Use the global keyword before var if creating a variable in a function,if,else,while and you want it to be accesible from outside.

4. If Else

To create a if statement, use the if keyword followed by the condition inside parantheses. Ex: if(x==y){Terminal.IO.print("x is equal to y.")} The else keyword can only be used after a if statement. The statements inside else only happen if the condition of the if statement is false. Ex: if(2==1){Terminal.IO.println("Correct")}else{Terminal.IO.println}

5. Operators

Comparison operators: is,==,<,>,<=,>=,!= (is and == have the same meaning). Logical operators: ||,&&,and,or (||=or,&&=and). Unary operators: !,-,+ (in some conditions the - and + are unary). Binary Operators: +,-,*,/,^ .

6. Casting

You can cast to int using Int(varname). You can cast to a double value using Double(varname). If you want to cast to a string, just do this. var x = 3; x = ""+3;

7. Syntax

You can either use semicolons or to seperate commands or you can just write it in the next line. Every block of code generates a new scope, but just curly parantheses {} wont generate a new scope.

8. Comments

You can make command using //. Ex: //this is a command!. Note: Multiline comments haven't been implemented yet but will be implemented.

9. Parameters

Parameters can now be used. The parameters type must be defined explicitly. Ex: fn test(string name){Terminal.IO.println(name);} test("John");

How to interpret graphite code?

REPL

To interpret graphite code, go to the graphite github page. If you want to have a repl, compile using this command:

g++ -Ofast -march=native -flto -fno-plt -fomit-frame-pointer lexer.cpp parser.cpp interpreter.cpp repl.cpp -o repl -lreadline

In the repl, you can enter --debug to see execution times. If you want to close debug mode, type --debug again.


Interpret whole files

To interpret whole files, use this command to compile the file interpreter:

g++ -Ofast -march=native -flto -fno-plt -fomit-frame-pointer lexer.cpp parser.cpp interpreter.cpp cliinterpret.cpp -o interpret

Usage: ./interpret code.gh
